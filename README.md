# Harsha Kaptures - Photography Portfolio Website

A clean, standalone photography portfolio website recreated from the Wix template, with no dependencies on Wix or external services.

## 📁 Directory Structure

```
harsha-kaptures-website/
├── index.html          # Main HTML file
├── styles.css          # All styles and layout
├── script.js           # JavaScript for interactivity
├── images/             # Image assets
└── README.md           # This file
```

## ✨ Features

- **Clean HTML Structure** - No Wix dependencies
- **Responsive Design** - Works on all devices
- **Smooth Scrolling** - Navigation with smooth scroll effects
- **Interactive Gallery** - 9 project images with hover effects
- **Contact Form** - Fully functional form (frontend ready)
- **Parallax Effects** - Subtle parallax on hero section
- **Modern Design** - Matches the original Wix template design

## 🚀 How to Use

1. **Open the website**: Simply open `index.html` in your web browser
2. **No server required**: Works directly from the file system
3. **All assets included**: Images are already in the `images/` folder

## 📋 Sections

1. **Hero Section** - Logo "HK", navigation, tagline, and scroll indicator
2. **Projects Gallery** - 3x3 grid of photography projects
3. **Pattern Section** - Decorative diagonal stripe pattern
4. **About Section** - Information about Harsha Kaptures
5. **Contact Image** - Subway station photograph
6. **Contact Section** - Contact form, email, and social media links

## 🎨 Customization

### Colors
Edit color values in `styles.css`:
- Text color: `#000` (black)
- Background: `#fff` (white)
- Hover effects: `rgba(0, 0, 0, 0.6)`

### Content
- Edit text directly in `index.html`
- Update images by replacing files in `images/` folder
- Modify form fields in the contact form section

### Form Submission
The contact form currently shows an alert on submission. To make it functional:
1. Set up a backend server (Node.js, PHP, Python, etc.)
2. Update the form handler in `script.js`
3. Or use a service like Formspree, Netlify Forms, or EmailJS
